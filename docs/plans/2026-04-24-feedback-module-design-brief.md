# Модуль «Обращения граждан» — дизайн-бриф для Claude Design

**Цель:** получить отрисованный редизайн всего модуля обращений граждан в АИС Минприроды КР — от подачи гражданином до обработки сотрудником.

---

## Контекст

**Система:** АИС Минприроды Кыргызской Республики (регулятор обращения с отходами). Правительственный портал.

**Модуль «Обращения граждан»** — позволяет гражданам подавать обращения (жалобы, предложения, запросы информации, сообщения о нарушениях), а сотрудникам МПРЭТН — обрабатывать их в ЛК.

**Законодательный контекст:** Закон КР № 195 от 19.10.2023 обязывает отвечать на обращения в течение **30 дней**.

---

## Пользователи и роли

### Гражданин (публичная сторона)
- Заходит на публичный сайт, заполняет форму обращения
- Получает номер тикета (например `OBR-2026-00002`)
- Может проверить статус своего обращения по номеру + email

### Сотрудник МПРЭТН (ЛК)
- Видит список всех обращений в своём кабинете
- Берёт обращения в работу или перенаправляет в другие отделы
- Отвечает гражданину, меняет статус, закрывает обращение

---

## Бизнес-логика: статусы и переходы

| Статус | Описание |
|---|---|
| **NEW** (Новое) | Только что подано, никто не взял в работу |
| **IN_PROGRESS** (В работе) | Сотрудник взял в работу, рассматривает |
| **UNDER_REVIEW** (На уточнении) | Либо перенаправлено в другой отдел, либо запрошено уточнение у гражданина |
| **RESOLVED** (Решено) | Ответ отправлен гражданину, обращение закрыто |
| **REJECTED** (Отклонено) | Обращение отклонено (например, не по компетенции) |

Переходы:
- Гражданин подаёт → `NEW`
- Сотрудник кликает «Взять в работу» → `IN_PROGRESS`
- Сотрудник кликает «Перенаправить в другой отдел» → `UNDER_REVIEW` + внутренний комментарий с деталями перенаправления
- Сотрудник в режиме работы: «Отправить ответ и закрыть» → `RESOLVED` (с заполненным `response` полем)
- Сотрудник: «Запросить уточнение у гражданина» → `UNDER_REVIEW` (с комментарием)
- Из `UNDER_REVIEW`: «Продолжить рассмотрение» → `IN_PROGRESS`
- Из `RESOLVED`/`REJECTED`: «Переоткрыть» → `IN_PROGRESS`

---

## Категории обращений

- **complaint** — Жалоба (например, на несанкционированную свалку)
- **suggestion** — Предложение (улучшение системы обращения с отходами)
- **info_request** — Запрос информации (как получить лицензию, и т.д.)
- **violation_report** — Сообщение о нарушении (юрлицом)

## Экраны для отрисовки

### Экран 1: Публичная форма подачи обращения (гражданин)
- URL: `/feedback`
- Поля: ФИО, телефон, email, категория (выпадашка), регион, тема, текст обращения, прикреплённые файлы (до 5), координаты (опционально — «Указать на карте»)
- Кнопка «Отправить обращение»
- После отправки — экран подтверждения с номером тикета и инструкцией «запишите номер»
- Должна быть ссылка «Проверить статус моего обращения»

### Экран 2: Проверка статуса (гражданин)
- URL: `/feedback/status`
- Ввод: номер тикета + email
- Если совпадение: показывает статус, текст ответа (если есть), историю изменений

### Экран 3: Список обращений (сотрудник МПРЭТН)
- URL: `/employee/feedback`
- Шапка: заголовок + кнопка «Экспорт в Excel»
- 4 stat-карточки: Всего / Новых / В работе / Решено
- Фильтры: поиск по номеру/ФИО/теме, статус, категория, период с/по
- Таблица: № тикета, дата, категория, тема, заявитель, регион, статус (бейдж), SLA (дней прошло), стрелка ›
- Клик по строке → детальная страница

### Экран 4: Детальная страница обращения (сотрудник) ⭐ ГЛАВНЫЙ
- URL: `/employee/feedback/:id`
- **Двухколоночный layout:**

  **ЛЕВАЯ колонка (2/3 ширины) — данные обращения (read-only):**
  - Карточка «Информация о заявителе» — ФИО, телефон, email, регион
  - Карточка «Содержание обращения» — категория (бейдж), тема, текст, вложения, координаты/адрес (если есть)
  - Карточка «История изменений» — timeline с статусами, датами, авторами, комментариями к переходам
  - Карточка «Внутренние комментарии» — чат-подобный список комментариев (видны только сотрудникам), поле ввода нового

  **ПРАВАЯ колонка (1/3 ширины) — action panel (меняется по статусу):**

  Главная карточка «Что делать сейчас»:
  - При `NEW`: две кнопки — «Взять в работу» (primary, brand color) + «Перенаправить в другой отдел» (secondary). Информационный блок: «Пока обращение не в работе, отвечать нельзя».
  - При `IN_PROGRESS`: textarea «Ответ гражданину» + кнопка «Отправить ответ и закрыть» (primary). Ниже: textarea «Запросить уточнение» + кнопка.
  - При `UNDER_REVIEW`: карточка «Ожидаем от гражданина / перенаправлено» + кнопка «Продолжить рассмотрение».
  - При `RESOLVED`: сводка «Обращение решено» + дата ответа + кнопка «Переоткрыть».
  - При `REJECTED`: сводка «Отклонено» + кнопка «Переоткрыть».

  Вторая карточка «Срок ответа по закону»:
  - «30 дней с момента подачи»
  - Индикатор: `●` (цвет: зелёный/жёлтый/красный в зависимости от SLA) + «Прошло: N дн. · Осталось: M дн.»

  Третья карточка «Исполнитель» (если назначен): имя сотрудника

### Экран 5: Модалка «Перенаправить в другой отдел»
Всплывающее окно:
- Заголовок «Перенаправить обращение»
- Список радиокнопок с отделами:
  - ГП «Эко Оператор» (вопросы переработки)
  - Местные органы власти
  - Санитарно-эпидемиологическая служба
  - Органы внутренних дел
  - Другой отдел / ведомство
- Textarea «Комментарий (опционально)»
- Кнопки «Отмена» / «Перенаправить»

## Визуальный стиль — требования

**Язык:** русский (интерфейс). Дополнительно — кыргызский и английский (через i18n, не критично для макетов).

**Ощущение:** официальное, госорган, строго, но современно. НЕ коммерческое/маркетинговое. НЕ игривое. Консерватизм + удобство.

**Палитра:**
- Бренд: зелёный `#2D8B4E`, бирюзовый `#0e888d`
- Нейтральные: slate-серые (`#f8fafc`, `#e2e8f0`, `#475569`, `#1e293b`)
- Семантика: красный для ошибок/просрочек, жёлтый для предупреждений, зелёный для успеха/решено, синий для новых
- Фон страниц: `#F8FAFC` (почти белый)

**Типографика:** системный стек (sans-serif). Без экстравагантных шрифтов. Размеры: заголовки 20-24px, основной текст 13-14px.

**Радиусы:** `8-16px` (`rounded-lg` / `rounded-xl`), мягкие тени.

**НЕ использовать:**
- Emoji как декоративные элементы (✓, ℹ️ и т.д.) — заменять SVG-иконками
- Яркие цветные баннеры (кроме ошибок/алертов)
- Мультиколор gradients
- Декоративные иллюстрации

**Компактность:** плотность важна. Сотрудник проведёт на этих экранах 8 часов в день. Чем меньше воздуха и лишних кликов — тем лучше.

## Данные модели (для понимания)

```ts
interface FeedbackItem {
  id: number
  ticketNumber: string           // 'OBR-2026-00002'
  createdAt: string              // ISO date
  updatedAt: string
  status: 'new' | 'in_progress' | 'under_review' | 'resolved' | 'rejected'
  category: 'complaint' | 'suggestion' | 'info_request' | 'violation_report'
  fullName: string
  phone: string
  email: string
  region?: string
  subject: string
  message: string
  attachments: { id: number; fileName: string; fileSize: number; fileType: string }[]
  latitude?: number
  longitude?: number
  locationAddress?: string
  timeline: {
    id: string
    timestamp: string
    status: FeedbackStatusType
    author: string
    comment?: string
  }[]
  comments: {
    id: string
    timestamp: string
    author: string
    authorRole: 'citizen' | 'employee'
    text: string
    internal: boolean              // true = виден только сотрудникам
  }[]
  response?: string
  respondedAt?: string
  assignedTo?: string
  assignedName?: string
}
```
