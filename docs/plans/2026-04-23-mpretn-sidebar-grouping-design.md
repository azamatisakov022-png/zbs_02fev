## Проблема

Сайдбар личного кабинета сотрудника МПРЭТН содержит 14 пунктов меню, расположенных плоским списком без логической группировки. Это создаёт когнитивную перегрузку: сотруднику трудно быстро найти нужный раздел, особенно новому. Пользователь описал состояние как «бардак».

Дополнительно: пункт «Выявленные компании» не нужен в сайдбаре сотрудника МПРЭТН.

## Область задачи (scope)

**В scope:**
- Визуальная группировка пунктов сайдбара в 4 логические секции с текстовыми заголовками
- Удаление пункта «Выявленные компании» из сайдбара
- Визуальное отделение «Уведомлений» и «Моего профиля» от основных разделов

**НЕ в scope (отложено):**
- Переделка дашборда (виджеты, KPI, Quick Actions)
- Перенос «Уведомлений» и «Профиля» в хедер (колокольчик + аватар-dropdown)
- Унификация путей `/ministry/*` → `/employee/*` (технический долг)
- Изменение функционала разделов
- Перестановка разделов по частоте использования

**Обоснование scope:** минимальное изменение, решающее главную жалобу (трудность ориентации), без риска сломать работающий функционал. Дальнейшие улучшения — по обратной связи от реальных сотрудников.

## Решение

### Новая структура сайдбара

```
🏠 Главная

ЗАЯВЛЕНИЯ И ОБРАЩЕНИЯ
  📄 Лицензии
  💬 Обращения граждан
  ✅ Контроль исполнения

РЕЕСТРЫ
  🗑️ Полигоны ТБО
  📍 Пункты приёма
  ♻️ Виды отходов

АНАЛИТИКА
  📊 Дашборд аналитики
  📑 Отчётность
  🗺️ ГИС-карта

СИСТЕМА
  📋 Журнал действий

─────────────────
  🔔 Уведомления  [badge]
  👤 Мой профиль
```

### Принципы группировки

1. **«Заявления и обращения»** — работа с входящими потоками от заявителей/граждан. Эпизодическая работа, но требующая конкретных действий.
2. **«Реестры»** — справочные данные об объектах обращения с отходами. Чтение и актуализация.
3. **«Аналитика»** — агрегированные данные, отчёты, пространственные визуализации. Работа «на большом масштабе».
4. **«Система»** — служебные разделы (журнал действий).

### Визуальное оформление заголовков секций

- Текст: **UPPERCASE**, мелкий шрифт (11-12px), цвет приглушённый серый
- **НЕ кликабельны** — это подписи-разделители, не кнопки и не аккордеоны
- Отступ сверху — обычный; снизу — небольшой
- Между секциями — обычный вертикальный интервал (без видимых линий-разделителей)

### Блок «Уведомления + Профиль»

- Отделён от основных секций **тонкой горизонтальной линией**
- Расположен **внизу сайдбара** (либо `margin-top: auto` в flex-контейнере)
- Сохраняет текущую функциональность (бейдж количества непрочитанных для уведомлений)

### Удаление «Выявленных компаний»

Пункт `Выявленные компании` / `/employee/detected-companies` удаляется из массива пунктов сайдбара. **Маршрут и страница остаются** — они могут быть доступны по прямой ссылке из других мест системы. Удаляется только ссылка из навигации.

## Технические детали

### Стек проекта

Проект — **Vue 3 + TypeScript** (не React). Сайдбар — это `.vue` компонент, конфигурация пунктов меню — в Vue composable.

### Файлы, которые будут затронуты

Ожидаемые кандидаты (уточнить на этапе имплементации):
- **`frontend/src/composables/useRoleMenu.ts`** — конфиг пунктов меню по ролям (точка, где живёт массив items для роли employee)
- **`frontend/src/components/dashboard/DashboardLayout.vue`** — layout с сайдбаром, где рендерятся пункты

Эти два файла уже есть в списке модифицированных в main — значит работа над сайдбаром уже в процессе, наши изменения туда же и лягут.

### Модель данных навигации

Текущая модель — плоский массив объектов `{ label, to, icon, badge? }`. Новая модель:

```ts
type NavSection = {
  title?: string;  // undefined для "Главная" и нижнего блока — нет заголовка
  items: NavItem[];
};

type NavItem = {
  label: string;
  to: string;
  icon: Component;
  badge?: () => number;  // функция или реактивная ссылка на счётчик
};

// Сигнатура функции useRoleMenu становится:
// useRoleMenu(role: Role) => ComputedRef<NavSection[]>
```

Сайдбар рендерит массив `NavSection[]` с условным рендерингом заголовка, если `title` задан.

### Что НЕ меняется

- Логика роутинга
- Права доступа к разделам
- Иконки (оставляем текущие)
- Бейджи и их значения
- Активное состояние пункта меню (подсветка текущего)

## Критерии приёмки

- [ ] Сайдбар визуально разбит на 4 секции + блок «Главная» сверху + блок «Уведомления/Профиль» внизу
- [ ] Заголовки секций стилизованы как подписи (UPPERCASE, мелкие, серые, некликабельные)
- [ ] Пункт «Выявленные компании» не отображается в сайдбаре
- [ ] Все остальные пункты меню сохранены и ведут на те же маршруты, что и раньше
- [ ] Бейдж непрочитанных уведомлений работает
- [ ] Активный пункт меню подсвечивается корректно
- [ ] Сайдбар корректно отображается на разных разрешениях (mobile, desktop)
- [ ] Существующие e2e-тесты, если они трогают сайдбар, обновлены

## Риски

1. **Сломаем тесты, которые ищут пункты по индексу в плоском списке.** Митигация — обновить селекторы на text-based.
2. **Визуальная регрессия на мобильных.** Митигация — проверить вручную в браузере на разных разрешениях перед мёрджем.
3. **Параллельные изменения в `useRoleMenu.ts` и `DashboardLayout.vue` в main.** Эти файлы уже модифицированы (uncommitted в main). Митигация — перед имплементацией посмотреть `git diff` этих файлов, согласовать изменения.
4. **Пользователи, которые привыкли к «Выявленным компаниям» в сайдбаре.** Митигация — сейчас проект ещё не в проде с реальными пользователями (контекст: госпроект, МПРЭТН), риск низкий.

## Дальнейшие шаги (не в этой итерации)

- Собрать обратную связь от реальных сотрудников после запуска
- На основе реальных паттернов использования — переставить пункты внутри секций по частоте
- Рассмотреть перенос уведомлений/профиля в хедер
- Почистить дашборд (убрать декоративные виджеты, усилить «Требует вашего действия»)
- Унифицировать пути `/ministry/*` → `/employee/*`
