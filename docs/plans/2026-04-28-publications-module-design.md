# Раздел «Публикации» — дизайн-бриф

**Дата:** 2026-04-28
**Автор:** AIS team
**Статус:** черновик · согласование с заказчиком
**Связанная задача:** редизайн `/publications` + админка для МПРЭТН и Эко-Оператора

---

## 1. Цель

Превратить статичную хардкод-страницу `/publications` (8 карточек в коде, дубли, не кликабельные)
в полноценный **новостной модуль АИС**: с БД-таблицей, CRUD-API,
ролевой админкой, многоязычностью и журнальным визуалом.

Раздел должен:
- Быть **единой точкой публикации** для МПРЭТН и Эко-Оператора по своим зонам ответственности.
- Дать читателю прозрачно понимать **«от какой организации новость»**.
- Поддерживать публикацию **на 3-х языках** (RU / KY / EN) — для гос-контента это требование.
- Выглядеть как современный editorial-блог (Stripe Press / Apple Newsroom-стиль),
  без «ИИ-шных» паттернов (gradient-абстракций, emoji-иконок, шаблонных «откройте для себя»).

---

## 2. Зоны ответственности

В АИС двойственное управление: МПРЭТН и Эко-Оператор — обе работают через систему,
но за разные участки. Распределяем права на публикации **по их реальной зоне**.

| Категория | МПРЭТН (Employee) | ГП Эко-Оператор | Админ |
|---|:---:|:---:|:---:|
| **Законодательство** (ПКМ, законы, регламенты МПРЭТН) | ✅ | — | ✅ |
| **Конкурсы** (гранты на эко-проекты) | — | ✅ | ✅ |
| **Отчёты** | ✅ министерства | ✅ по утильсбору | ✅ |
| **Аналитика** | ✅ экология/мониторинг | ✅ утильсбор/ставки | ✅ |
| **Новости** | ✅ реестры, лицензии, ГИС | ✅ калькулятор, ставки | ✅ |
| **Объявления** | ✅ полигоны, лицензии, регламент | ✅ сроки, изменения | ✅ |
| **Пресс-релизы** | ✅ от Минприроды | ✅ от ГП «Эко Оператор» | ✅ |

**Ключевые принципы:**
- В форме создания категории фильтруются по роли: пользователь не видит запрещённых опций
  (вместо «у вас нет прав» — просто чистый UI).
- Каждая публикация хранит **`author_org`** (`MPRETN` / `ECO_OPERATOR` / `ADMIN_GENERIC`).
  На карточке отображается лейбл «🏛 МПРЭТН КР» / «🌱 ГП „Эко Оператор"».
- **Без модерации на старте.** Employee и Эко-Оператор публикуют сразу.
  Если потом возникнет необходимость workflow — добавим `draft → review → published`.
- **Админ может править/снять с публикации/удалить ЛЮБУЮ публикацию** независимо от автора.

## 3. Data model

### Таблица `publications`

| Поле | Тип | Описание |
|---|---|---|
| `id` | BIGSERIAL PK | |
| `slug` | VARCHAR(255) UNIQUE | URL-фрагмент `/publications/<slug>` |
| `category` | VARCHAR(20) NOT NULL | enum: `news`, `report`, `legislation`, `contest`, `announcement`, `analytics`, `press_release` |
| `author_org` | VARCHAR(20) NOT NULL | `MPRETN` / `ECO_OPERATOR` / `ADMIN_GENERIC` |
| `cover_url` | VARCHAR(500) NOT NULL | путь в MinIO (`publications/{id}/cover.jpg`) — **обязательно** |
| **Многоязычные поля** | | |
| `title_ru`, `title_ky`, `title_en` | VARCHAR(500) | `_ru` обязательное, остальные null → fallback на ru |
| `excerpt_ru`, `excerpt_ky`, `excerpt_en` | TEXT | короткий лид (150-200 символов) |
| `body_ru`, `body_ky`, `body_en` | TEXT | основной HTML/Markdown |
| **Метаданные** | | |
| `is_published` | BOOLEAN DEFAULT TRUE | снять с публикации, не удаляя |
| `published_at` | TIMESTAMP | момент первой публикации |
| `created_by_id` | BIGINT REFERENCES users(id) | кто создал |
| `last_edited_by_id` | BIGINT REFERENCES users(id) | кто последний редактировал |
| `unpublished_by_id` | BIGINT NULL | кто снял с публикации (если снимали) |
| `unpublished_at` | TIMESTAMP NULL | |
| `view_count` | INTEGER DEFAULT 0 | счётчик просмотров (опционально) |
| `created_at`, `updated_at` | TIMESTAMP | |

**Индексы:**
- `(is_published, published_at DESC)` — для главной страницы (фильтр+сортировка)
- `(category, is_published, published_at DESC)` — для фильтра по категории
- `(author_org, is_published, published_at DESC)` — для фильтра по автору
- `slug` UNIQUE

**Без отдельной таблицы переводов** (3 языка × 3 поля = 9 колонок — терпимо
и проще, чем нормализованная схема).

### Cover-картинки в MinIO

- bucket: `eco-operator` (текущий)
- путь: `publications/<id>/cover_<size>.<ext>`
- размеры: `original` (загружено), `cover` (1200×675, для hero), `thumb` (600×338, для карточек)
- генерация миниатюр — на сервере при загрузке (Spring Boot с Thumbnailator)

## 4. API

### Публичные (без auth)

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/v1/public/publications` | Список published, поддержка фильтра+пагинации |
| `GET` | `/api/v1/public/publications/{slug}` | Деталь по slug |

**Параметры списка:**
```
?category=news,report&authorOrg=ECO_OPERATOR&search=утильсбор&lang=ru
&page=0&size=12&sort=published_at,desc
```

Ответ: страница с `total`, `content`, `page`, `size`. Каждая запись — урезанный DTO
(без `body_*`, чтобы не тащить мегабайты): id, slug, category, authorOrg, cover_thumb_url,
title (на запрошенном lang с fallback ru), excerpt, publishedAt.

### Админка (auth)

| Метод | Путь | Роль | Описание |
|---|---|---|---|
| `GET` | `/api/v1/publications/my` | EMPLOYEE / ECO_OPERATOR / ADMIN | свои публикации (для ADMIN — все) |
| `POST` | `/api/v1/publications` | EMPLOYEE / ECO_OPERATOR / ADMIN | создать |
| `PUT` | `/api/v1/publications/{id}` | автор или ADMIN | редактировать |
| `POST` | `/api/v1/publications/{id}/cover` | автор или ADMIN | загрузить обложку (multipart) |
| `POST` | `/api/v1/publications/{id}/publish` | автор или ADMIN | опубликовать (если был unpublished) |
| `POST` | `/api/v1/publications/{id}/unpublish` | автор или ADMIN | снять с публикации |
| `DELETE` | `/api/v1/publications/{id}` | только ADMIN | удалить совсем |

**Валидация на сервере:**
- `category` обязательно валидно для роли автора (Эко-Оператор не может постить в `legislation`).
- `cover_url` обязательное.
- `title_ru`, `body_ru` обязательные (русский — основной язык).

## 7. Визуальная концепция — Magazine-Hybrid

### Принципы

| Правильно | Неправильно («ИИ-стиль») |
|---|---|
| Реальные cover-картинки | Generic gradient-абстракции purple-pink |
| Цветной маркер 8px рядом с категорией | Эмодзи-иконки 🆕📊🏆 |
| Иерархия: hero > большая карточка > обычная | Все карточки одинаковые |
| Конкретные числа («42.3 млн с.», «12 450 т») | «Узнайте больше», «Откройте для себя» |
| Метаданные мелким мутным шрифтом | Метаданные навязчивым brand-цветом |
| Автор-органзизация на каждой карточке | Безликие посты «от системы» |

### Палитра

Используем существующую teal/emerald `--lr-brand: #10b981`, `--lr-brand-700: #0f766e`.
**НЕ вводим новые hue-абстракции.** Каждая категория получает 1 цвет-маркер
(только маленький кружок 8px рядом с надписью):

| Категория | Цвет маркера |
|---|---|
| Законодательство | `#0f766e` (teal-700) |
| Конкурсы | `#f59e0b` (amber-500) |
| Отчёты | `#0d9488` (teal-600) |
| Аналитика | `#6366f1` (indigo-500) |
| Новости | `#10b981` (emerald-500) |
| Объявления | `#64748b` (slate-500) |
| Пресс-релизы | `#475569` (slate-600) |

Цвет — **только маркер**, не фон. Карточки белые с тонкой границей `--lf-line`.

### Типографика

- Тело: **Inter** (уже подключён) — 14-16px
- Заголовки на главной: Inter Bold 18-22px
- **На детальной странице** заголовок может быть **Playfair Display 36-44px serif**
  для журнального ощущения (только h1, остальные h2/h3 — Inter)
- Метаданные: 11-12px ink-4 uppercase letter-spacing 0.4

### Авторские лейблы

```
🏛 МПРЭТН КР      — иконка building, цвет ink-2 #2b3a34
🌱 ГП Эко Оператор — иконка leaf, цвет brand-700 #0f766e
```

Иконки — линейные SVG (Lucide / Heroicons), **не emoji**.

## 8. Анимации (spec)

Анимации — **сдержанные, тонкие, заметны но не отвлекают**. Никакой parallax,
spring-bounces, particle-effects.

| Где | Что | Тайминг | Easing |
|---|---|---|---|
| Карточки на главной | Stagger fade-up при скролле в viewport | 200ms на карточку, delay 50ms между | `cubic-bezier(0.2, 0.8, 0.2, 1)` |
| Hover на карточке | translateY(-2px) + box-shadow усиливается | 150ms | `ease-out` |
| Cover-картинка | Blur-up: загружается LQIP placeholder, потом fade в чёткое | 200ms | `ease-out` |
| Hero на детальной | Subtle scroll-progress bar сверху страницы | реалтайм | linear |
| Page transitions | Fade-in основного контента (не slide, не scale) | 180ms | `ease-out` |
| Tabs / категории | Underline-animation при переключении | 120ms | `ease-in-out` |
| Поиск | Debounce 250ms на input + skeleton-карточки во время загрузки | — | — |
| Skeleton-loading | Shimmer-анимация на placeholder-карточках | 1500ms loop | linear |

**Доступность:** все анимации респектят `prefers-reduced-motion`. При включённой настройке
ОС — анимации отключаются (только мгновенные fade без motion).

**Реализация:** CSS animations + Vue `<Transition>` где можно. Для stagger fade-up — IntersectionObserver
+ data-attributes (без библиотек). Не тащим **GSAP / Framer Motion** — overhead не оправдан.

## 9. Многоязычность

Подключение к существующему `vue-i18n` (RU/KY/EN).

- Список языков в публикациях == системные локали
- На главной странице: **тексты публикаций отдаются на текущей локали** с fallback на `ru` если перевод пуст
- На детальной странице: **переключатель** языков для конкретной статьи
- В админке при создании: **табы** RU/KY/EN, RU обязателен, KY/EN — опциональны
- В таблице админки видно **значки языков**, на которых публикация переведена
  (например `RU · KY` — английский ещё не написан)

## 11. Что НЕ делаем (YAGNI)

- ❌ Комментарии под публикациями — не гос-формат
- ❌ Лайки / реакции — то же
- ❌ Подписка по email — отдельный модуль если потребуется
- ❌ Категории-теги (свободные ярлыки) — у нас 7 жёстких категорий, этого хватит
- ❌ Multi-author (несколько авторов одной статьи) — usecase редкий
- ❌ Версионирование (история редакций) — пока не требуется
- ❌ Workflow draft → review — добавим если возникнет реальная потребность

- [Stripe Press](https://press.stripe.com/) — magazine layout, типографика
- [Apple Newsroom](https://www.apple.com/newsroom/) — hero-публикация + сетка
- [Vercel Blog](https://vercel.com/blog) — анимации, hover-эффекты
- [GOV.UK news](https://www.gov.uk/government/news) — институциональная подача (что **не** копируем буквально, но тон подобен)

### B. Текущее состояние

- `frontend/src/views/PublicationsView.vue` — 268 строк, 8 хардкод-карточек
- `frontend/src/views/publications/WasteLifetime.vue` — 1202 строки, спецпубликация (live)
- `/admin/*` — нет страниц публикаций, нужно добавить
- БД — таблицы `publications` нет, нужна миграция

### C. Стек

- Backend: Spring Boot 3 + JPA + Flyway + MinIO (для cover-картинок)
- Frontend: Vue 3 + TS + Vue Router + vue-i18n + Tailwind + scoped CSS
- Editor: **Tiptap** (выбран, лёгкий, расширяемый, MIT)
- Image-thumbs: **Thumbnailator** на бэке (Java)
