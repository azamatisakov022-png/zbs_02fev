## 5. Сквозные сценарии

### 5.1 Полный цикл расчёта
1. [ ] Плательщик создаёт расчёт
2. [ ] Плательщик отправляет расчёт → Эко Оператор получает уведомление
3. [ ] Эко Оператор одобряет расчёт → Плательщик получает уведомление
4. [ ] Плательщик загружает платёжный документ
5. [ ] Эко Оператор подтверждает оплату → Плательщик получает уведомление

### 5.2 Полный цикл отчёта
1. [ ] Переработчик создаёт отчёт
2. [ ] Переработчик отправляет отчёт → Эко Оператор получает уведомление
3. [ ] Эко Оператор возвращает на доработку → Переработчик получает уведомление
4. [ ] Переработчик исправляет и повторно отправляет
5. [ ] Эко Оператор одобряет отчёт → Переработчик получает уведомление

### 5.3 Полный цикл возврата
1. [ ] Плательщик создаёт заявку на возврат
2. [ ] Эко Оператор рассматривает заявку
3. [ ] Эко Оператор одобряет/отклоняет → Плательщик получает уведомление
